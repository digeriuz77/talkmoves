# Dialogic Classroom UX/UI Improvements and Simplified Scenarios Plan

## Executive Summary
This plan addresses two main issues:
1. UX/UI improvements to make the app mobile and laptop friendly
2. Creation of new simplified scenarios for basic English comprehension (Year 1 Maths focus)

## 1. Current UI/UX Issues Analysis

### Problems Identified:
- Fixed positioning and sizing that doesn't adapt to different screen sizes
- Text elements with fixed pixel sizes that don't scale
- Touch targets that may be too small for mobile interaction
- Layouts that assume desktop aspect ratios
- Complex game controls that may be overwhelming on touch screens

### Key Components Reviewed:
- App.tsx: Main application structure with flexbox layouts
- Game.tsx: Core game logic with fixed positioning elements
- TalkMovesGame.tsx: Chain-building interface with grid layouts
- Classroom.tsx: Visual classroom representation with percentage-based positioning
- index.css: Tailwind CSS imports (responsive by default but needs customization)

## 2. Specific Responsive Design Improvements

### CSS/Scaling Improvements:
1. Replace fixed pixel values with relative units (rem, em, %) where appropriate
2. Implement proper breakpoints using Tailwind's responsive prefixes
3. Ensure minimum touch target sizes (48x48px) for mobile
4. Scale text appropriately for readability on small screens
5. Adjust grid layouts to collapse appropriately on narrow screens

### Specific Component Fixes:
- GameSessionHeader: Make title/subtitle text scale properly
- DialogueBox: Ensure choice buttons are appropriately sized for touch
- Talk moves grid: Adjust column counts based on screen width
- CoachingStrip: Make dismiss button easily tappable
- Classroom visualization: Ensure student hotspots remain usable

### Implementation Approach:
1. Add responsive utility classes to existing components
2. Create mobile-specific variants where layouts fundamentally change
3. Test with actual device dimensions or responsive design tools

## 3. New Simplified Scenarios for Year 1 Maths Basic Comprehension

### Target Learning Objectives:
- Basic counting (1-20)
- Simple addition/subtraction within 10
- Shape recognition (circle, square, triangle)
- Basic measurement concepts (more/less, bigger/smaller)
- Pattern recognition (ABAB, AABB)

### Scenario Structure:
Each scenario will follow this pattern:
1. **Student Response Type**: Emergent language or partial idea (single words/phrases)
2. **Teacher Challenge**: Student gives minimal response requiring extension
3. **Available Talk Moves**: Focus on Wait Time, Say More, Revoicing, Add On
4. **Progression**: Build from single words to short phrases to complete thoughts

### Example Scenario: "Counting Blocks"
- **Setup**: Teacher shows varying numbers of blocks
- **Student Response**: "Three" (when asked how many)
- **Teacher Options**:
  - Wait Time (let them think)
  - Say More ("Can you tell me how you saw three?")
  - Revoicing ("So you saw three blocks?")
  - Add On ("Who can show another way to make three?")
- **Success Criteria**: Student extends beyond single word to explain reasoning

### Implementation Plan:
1. Create new scenario files in src/data/ (choice-scenarios.ts or talk_moves.ts)
2. Add to game catalog with appropriate recommendedOrder
3. Develop appropriate student responses representing basic comprehension level
4. Map to existing talk moves that support language development
5. Create appropriate visual assets or descriptions

## 4. Teacher Support Features for Better Explanations

### Current Support:
- Research tips on talk moves
- Feedback after selections
- Coaching strip with messages
- End screen advice based on performance

### Enhancements Needed:
1. **Contextual Prompt Suggestions**: 
   - Real-time suggestions based on student response type
   - Specific question stems for extending thinking ("What makes you think that?")
   
2. **Response Templates/Models**:
   - Examples of how to revoice student contributions
   - Demonstrations of wait time usage
   - Models for building on partial ideas

3. **Visual Wait Time Indicator**:
   - Subtle timer or visual cue when wait time is active
   - Feedback on whether sufficient pause was given

4. **Move Chaining Guidance**:
   - Suggestions for effective combinations (Wait Time → Say More)
   - Visual indication when a combo is possible
   - Explanation of why certain moves work well together

### Implementation:
1. Enhance the coaching strip to show contextual suggestions
2. Add template examples in the feedback system
3. Create visual indicators for wait time usage
4. Add subtle guidance for effective move combinations

### Phase 3: Teacher Support Enhancements (Week 1-2, overlapping)

- Improve feedback with modeling examples

### Phase 4: Testing and Refinement (Ongoing)

## 6. Success Metrics

### UX/UI Improvements:
- App usable on screens as small as 320px width
- All text readable without zooming
- All interactive elements easily tappable
- No horizontal scrolling on mobile

### Scenario Effectiveness:
- Teachers able to elicit extended responses from basic English speakers
- Clear progression from single words to explanatory phrases
- Appropriate challenge level for target age group

### Teacher Support:
- Increased use of wait time and revoicing moves
- Teachers reporting increased confidence in handling basic responses
- Observable improvement in student explanation quality

## 7. Dependencies and Risks

### Dependencies:
- Existing talk moves framework must remain compatible
- Visual assets may need creation or sourcing
- Testing requires access to target devices or emulators

### Risks:
- Over-simplifying scenarios might reduce educational value
- Too many UI changes might break existing functionality
- Balancing simplicity with engagement for teachers

### Mitigation:
- Maintain backward compatibility where possible
- Test changes incrementally
- Consult with educational experts on scenario design
- Use component-based approach to minimize disruption

This plan provides a comprehensive approach to making the Dialogic Classroom app more accessible and effective for its target users while maintaining its core educational value.
